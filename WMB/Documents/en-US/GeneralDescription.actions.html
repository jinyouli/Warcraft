This is a group of actions, each of which gets executed by in the top-down approach. Each action represents a certain unit of customizable behavior executed by GPS, which consists of a type, an optional argument and the finish conditions. Unless the finish conditions are met, the action will also get executed repeatedly.

If the actions reside under a waypoint, they will be run as soon as the character reaches the waypoint.

Right click the node to add a new action, with the following action expression syntax.

ACTION_TYPE([ACTION_ARGUMENT])

For example,

set_next_waypoint(5)
